# AGENTS.md

## コミットメッセージ規約

- AIエージェントがコミットを作成する場合、コミットメッセージは日本語で記述すること。
- AIエージェントがコミットを作成する場合、コミットメッセージは Conventional Commits の仕様に準拠するよう努めること。
- AIエージェントがコミットを作成する場合、コミットメッセージに `Assisted-by:` を含めること。
- `Assisted-by:` には、実行したAIエージェント自身のモデル名を含めること。

## Git運用ルール

- AIエージェントは、ローカル環境での変更作業とコミット作成まで実施してよい。
- `git pull`、`git push`、`git fetch` など、リモートリポジトリとの通信を伴う操作は人間のみが実施すること。

## Dev Container 共通ルール

### 基本方針

- この環境は比較的安全な開発環境であるため、AIエージェントはシェルコマンドの実行を許可される。
- ただし高い権限は一般に付与されていないため、権限を要する操作は失敗する可能性がある。

### 実行ポリシー

- 通常の開発コマンド（例: `npm test`, `git` など）は、原則として追加確認なしで実行してよい。
- Dockerコンテナ操作が必要な場合は、AIエージェントが開発者に実行を依頼する。

### 権限不足・実行失敗時の対応

- 権限不足や環境制約でコマンド実行に失敗した場合、AIエージェントは開発者に実行を依頼する。
- 依頼時は毎回、次の3点を必ず提示する。
  1. 実行しようとしたコマンド
  2. そのコマンドの目的（なぜ必要か）
  3. 開発者向けの実行手順
